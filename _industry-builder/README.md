# Turnkey AI — Industry Page Builder

Generates the 30 industry sub-pages, the industry hub, the blog hub, and the
SEO files (`sitemap.xml`, `robots.txt`) for the Turnkey AI marketing site.

Plain HTML/CSS/JS — no build step. Generated files are committed and served
directly by GitHub Pages.

## Files

| File | Purpose |
|---|---|
| `seed_industries.py` | Holds the content for all 30 trades. Writes `industries/<slug>.json`. |
| `industries/*.json`  | 30 per-industry data files (generated by the seed script). |
| `build_industries.py`| Renders every JSON → `../industries/<slug>/index.html`, plus the hub, blog, sitemap, robots. Patches the home nav/footer. |

## Regenerate

```sh
cd _industry-builder
python seed_industries.py     # only if you changed industry content
python build_industries.py    # always — renders all pages
```

`build_industries.py` is idempotent — re-running cleanly overwrites the
generated pages and re-applies (without duplicating) the home-page patch and
the appended CSS block in `../styles.css`.

## What gets written (all under the site root `../`)

- `industries/<slug>/index.html` — 30 industry pages
- `industries/index.html` — industry hub (grouped by category)
- `blog/index.html` + `blog/blog-styles.css` — blog hub (empty-state)
- `sitemap.xml`, `robots.txt`
- appends an `/* === INDUSTRY-BUILDER GENERATED STYLES === */` block to `styles.css`
- patches `index.html` nav + footer with Industries / Blog links

## Editing content

To change a trade's copy, edit the entry in `seed_industries.py`, then run both
scripts. Each industry entry needs: `slug`, `name`, `name_plural`, `group`,
`title`, `meta_description`, `h1_lead`, `h1_grad`, `hero_lede`, `drains` (list),
`tools` (3–4 `{tag,title,body}`), `cases` (1–2 `{context,before,after,impact}`),
`faq` (3 `{q,a}`).

## URL structure

The site is a GitHub Pages **project** site served under `/turnkey-ai-site/`.
All generated links are root-relative with that prefix (e.g.
`/turnkey-ai-site/industries/hvac/`). If a `CNAME` file is added (custom
domain), the builder auto-switches to domain-root paths — no code change needed.

## Blog handoff

`blog/index.html` ships in an empty-state. The SEO content engine publishes
articles at `blog/<slug>/index.html` and adds `<article class="post-card">`
entries inside `#blog-list`. Article-body styles already live in
`blog/blog-styles.css` (`.article`, `.article-body`, etc.).
